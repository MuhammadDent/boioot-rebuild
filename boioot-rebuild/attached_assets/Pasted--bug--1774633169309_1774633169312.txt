لدينا bug واضح في نظام الإشعارات داخل حساب المستخدم.

المشكلة الحالية:
عند الضغط على إشعار متعلق بطلب الاشتراك أو رد الأدمن على طلب الاشتراك، النظام لا يفتح تفاصيل الرد ولا يفتح الطلب المرتبط، بل يحول المستخدم فقط إلى صفحة "اشتراكي".
هذا سلوك غير صحيح.

المطلوب:
إصلاح نظام فتح الإشعارات بحيث يكون كل إشعار قابلاً لفتح المحتوى المرتبط به بشكل صحيح، وليس التحويل إلى صفحة عامة فقط.

==================================================
المطلوب الوظيفي الصحيح
==================================================

1) عند الضغط على إشعار من نوع:
- subscription_request_approved
- subscription_request_rejected
- subscription_request_needs_info
- subscription_request_reviewed
- admin_reply_on_subscription_request

يجب أن يفتح النظام أحد الخيارين التاليين:
- إما صفحة تفاصيل طلب الاشتراك المرتبط
- أو Modal / Drawer يعرض كامل رد الأدمن وتفاصيل الطلب

وليس صفحة "اشتراكي" العامة فقط.

2) يجب أن يرى المستخدم داخل تفاصيل الإشعار:
- عنوان الإشعار
- حالة الطلب الحالية
- نص رد الأدمن
- الملاحظة أو سبب الرفض أو الموافقة أو طلب الاستكمال
- تاريخ الرد
- اسم الباقة المطلوبة
- رقم أو معرف الطلب إن وجد

3) إذا كان الإشعار مرتبطًا بطلب اشتراك محدد، يجب أن يكون هناك ربط مباشر بالطلب نفسه عبر:
- requestId
أو
- entityId
أو
- targetId

4) إذا لم يكن هناك كيان مرتبط، افتح صفحة قائمة الإشعارات مع تحديد الإشعار المفتوح وإظهار كامل نصه.

==================================================
تحليل تقني متوقع للمشكلة
==================================================

راجع هذه النقاط وأصلحها:

A) Notification payload
- تأكد أن كل إشعار خاص بطلب الاشتراك يحتوي حقولًا مثل:
  - type
  - title
  - message
  - userId
  - targetType = "subscription_request"
  - targetId = subscriptionRequestId
  - actionUrl (optional)
  - metadata/adminReply/status/planName

B) Click handler
- راجع handler الخاص بالنقر على الإشعار في الـ header dropdown/mobile menu
- لا تجعل جميع إشعارات الاشتراك تذهب إلى route عام مثل:
  /dashboard/subscription
- اجعل التوجيه يعتمد على notification.type و notification.targetId

مثال السلوك المطلوب:
- إذا كان type من أنواع ردود الاشتراك و targetId موجود:
  انتقل إلى:
  /dashboard/subscription/requests/{targetId}
  أو افتح modal details لنفس الطلب
- إذا لا يوجد targetId:
  انتقل إلى:
  /dashboard/notifications/{notificationId}

C) Router
- أضف route حقيقي لتفاصيل طلب الاشتراك للمستخدم
- مثال:
  /dashboard/subscription-requests/:id
أو
  /dashboard/notifications/:id

D) Backend/API
- أضف endpoint يعيد تفاصيل طلب الاشتراك المرتبط بالإشعار
- وأعد أيضًا admin note / review note / decision reason

E) Notification read state
- عند الضغط على الإشعار:
  - علّمه كمقروء
  - ثم افتح التفاصيل الصحيحة
  - لا تجعل عملية mark as read تمنع navigation

==================================================
تنفيذ واجهة المستخدم
==================================================

أنشئ صفحة أو Modal باسم:
Subscription Request Details

وتعرض:
- عنوان الطلب
- اسم الباقة الحالية / المطلوبة
- نوع الحساب
- حالة الطلب:
  pending / approved / rejected / needs_info
- رد الأدمن
- ملاحظات الأدمن
- تاريخ الإنشاء
- تاريخ آخر تحديث
- زر "العودة للإشعارات"
- زر "الذهاب إلى الاشتراك" فقط كخيار ثانوي، وليس الوجهة الأساسية

==================================================
تحسين تجربة المستخدم
==================================================

داخل قائمة الإشعارات:
- الإشعارات المرتبطة بطلبات الاشتراك يجب أن تحمل action label واضح مثل:
  "عرض الرد"
  أو
  "عرض تفاصيل الطلب"

- إذا كان الإشعار من نوع approved/rejected/needs_info أظهر badge للحالة

==================================================
اختبارات مطلوبة
==================================================

نفّذ واختبر السيناريوهات التالية:

1. إشعار موافقة على طلب الاشتراك
- الضغط عليه يجب أن يفتح تفاصيل الرد والطلب الصحيح

2. إشعار رفض طلب الاشتراك
- الضغط عليه يجب أن يفتح سبب الرفض كاملًا

3. إشعار طلب استكمال نواقص
- الضغط عليه يجب أن يفتح ملاحظات الأدمن والنواقص المطلوبة

4. من dropdown الإشعارات في الموبايل
- يجب أن يعمل الفتح بشكل صحيح

5. من صفحة كل الإشعارات
- يجب أن يعمل الفتح بشكل صحيح

6. يجب ألا يتم فتح صفحة "اشتراكي" العامة إلا إذا ضغط المستخدم بنفسه على زر الاشتراك، وليس عند الضغط على الإشعار

==================================================
مخرجات مطلوبة
==================================================

أريد منك:
1. إصلاح كامل للمشكلة
2. توحيد سلوك فتح الإشعارات في الموبايل والديسكتوب
3. إنشاء صفحة/مودال تفاصيل رد الأدمن على طلب الاشتراك
4. ربط كل إشعار بطلبه الصحيح
5. تزويدي بأسماء الملفات التي تم تعديلها
6. شرح السبب الجذري للمشكلة Root Cause
7. التأكد أن الحل لا يكسر باقي الإشعارات
