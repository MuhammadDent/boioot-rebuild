We are now implementing Phase 3A of the Boioot SaaS platform billing system.

IMPORTANT CONTEXT:
We have an uploaded Excel pricing/plan matrix that must be treated as the initial source of truth for plan capabilities, limits, and feature mapping.
Use that workbook as a reference while implementing this phase.

The uploaded matrix includes:
- plan columns for seeker/researcher, owner, broker, office, company
- grouped feature sections
- feature keys
- value types such as Boolean, Limit, and text/manual policy-like values

This means the system must not assume every plan capability is only a boolean or numeric limit.
Some capabilities may be policy/text-based and should remain representable in the data model.

GOAL:
Build a real subscription core for plans/billing, so the system can assign plans to accounts, track subscription status, support upgrades/downgrades/cancellations, and prepare for future payment gateway integration.

IMPORTANT:
This is Phase 3A only.
Do NOT integrate a payment gateway yet.
Do NOT overbuild invoicing yet.
Do NOT implement limit enforcement yet (that will be Phase 3B).
In this phase we want the subscription data model, plan capability mapping, APIs, admin management, and user subscription visibility.

CURRENT CONTEXT:
- Auth system is complete
- CMS is complete
- Plans already exist
- Plan features/limits exist or partially exist
- We now need a proper subscription layer instead of plan display only
