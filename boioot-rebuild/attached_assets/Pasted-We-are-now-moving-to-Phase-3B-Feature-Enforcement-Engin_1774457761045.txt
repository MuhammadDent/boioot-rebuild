GOAL:

Enforce plan capabilities across the platform:
- features (boolean)
- limits (numeric)
- policies (access rules)

---

1) Create centralized access helper

Create a service:

ICurrentUserCapabilities

Methods:

- hasFeature(key)
- getLimit(key)
- canUseFeature(key)
- assertFeature(key)
- assertLimit(key, currentUsage)

---

2) Listing system enforcement

Apply rules:

- listing_publish → must be true
- max_active_listings → limit check
- image_gallery_enabled → allow images
- video_upload → allow video
- 360_view_enabled → allow 360

If violated:
- return proper error message
- do NOT allow operation

---

3) CRM / messaging enforcement

- max_conversations
- lead_tracking
- lead_assignment

Block or limit usage accordingly

---

4) Analytics enforcement

- analytics_dashboard
- advanced_reports

Hide or block endpoints if not allowed

---

5) Policy enforcement

VERY IMPORTANT:

Use policies:

- if policy = admin_only → block user action
- if policy = self_service → allow UI
- if policy = open → always allow

---

6) UI integration (light)

Frontend should:

- hide unavailable features
- disable buttons when limit reached
- show upgrade prompts

---

7) Error handling

Standardize errors:

- "Feature not available in your plan"
- "Limit reached: upgrade required"
- "This feature requires admin activation"

---

8) Logging (important)

Log:

- feature blocked
- limit exceeded
- policy denied
