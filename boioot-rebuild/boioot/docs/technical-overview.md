## أولاً: آلية عمل التطبيق والربط البرمجي

### 1.1 نظرة عامة على المنصة

**بيوت** (Boioot) منصة عقارية إلكترونية موجّهة للسوق السوري، تتيح للمستخدمين نشر واستعراض الإعلانات العقارية (شراء، بيع، إيجار، مشاريع). تعمل المنصة بنظام خدمتين منفصلتين متكاملتين:

| المكوّن | التقنية | الدور |
|---------|---------|-------|
| واجهة المستخدم (Frontend) | Next.js 16 — React / TypeScript | عرض الصفحات، إدارة تجربة المستخدم |
| خادم البيانات (Backend API) | ASP.NET Core 8 — C# | معالجة المنطق التجاري، REST API |
| قاعدة البيانات | SQLite عبر Entity Framework Core 8 | التخزين الدائم للبيانات |

```

│           المستخدم / المتصفح                    │

└─────────────────┬───────────────────────────────┘
                  │ HTTPS

                  │ HTTP / REST JSON
```

### 1.3 آلية المصادقة والتفويض

**نظام الهوية:** JWT (JSON Web Tokens)

| الخطوة | الآلية |
|--------|---------|
| **تسجيل الدخول** | يرسل العميل email + password → API يُعيد JWT (Access Token) + Refresh Token (HttpOnly Cookie) |
| **التحقق من الجلسة** | كل طلب يحمل `Authorization: Bearer <token>` في الرأس |
| **انتهاء الصلاحية** | Access Token قصير الأمد — يُجدَّد تلقائياً عبر Refresh Token |
| **الصلاحيات (RBAC)** | نظام صلاحيات مرن: أدوار + صلاحيات مخصصة لكل دور |

**أدوار المستخدمين:**

| الدور | الفئة | الوصف |
|-------|--------|-------|
| `Admin` | إداري | مدير النظام — صلاحيات كاملة |
| `User` | عميل | مستخدم عادي |
| `Owner` | عميل | مالك عقار |
| `Broker` | عميل | وسيط عقاري |
| `Office` | عميل | مكتب عقاري (كيان مستقل) |
| `CompanyOwner` | عميل | شركة تطوير عقاري |
| `Agent` | تابع | وكيل تابع لمكتب أو شركة |
| `AdminManager` | طاقم داخلي | مدير العمليات |
| `CustomerSupport` | طاقم داخلي | دعم العملاء |
| `ContentEditor` | طاقم داخلي | محرر المحتوى |

### 1.4 آلية التواصل بين الواجهة والـ API

```
Frontend (Next.js)          Backend (ASP.NET Core)
      │                              │
      │  GET /api/properties         │
      │─────────────────────────────►│
      │                              │  ← يتحقق من JWT
      │                              │  ← يستعلم قاعدة البيانات
      │  200 OK { data: [...] }      │
      │◄─────────────────────────────│
      │                              │
      │  POST /api/auth/login        │
      │─────────────────────────────►│
      │  { email, password }         │
      │                              │  ← يتحقق من كلمة المرور (BCrypt)
      │                              │  ← يُنشئ JWT
      │  200 OK { token, user }      │
      │◄─────────────────────────────│
```

**المتغيرات البيئية للربط:**
```
NEXT_PUBLIC_API_URL=/api
BACKEND_URL=http://localhost:5233
```

الواجهة تُوكّل (proxy) جميع طلبات `/api/*` إلى خادم ASP.NET على المنفذ 5233.

## ثالثاً: صورة المؤهل العلمي للمطور ومسؤول قواعد البيانات

> **خارج نطاق هذه الوثيقة التقنية.**  
>
> المؤهلات الأكاديمية والشخصية للمطورين ومسؤولي قواعد البيانات هي معلومات شخصية لا تندرج ضمن المستندات التقنية للنظام.  
>
> للحصول على هذه المعلومات، يُرجى التواصل مباشرة مع:  
> **مالك المشروع / الجهة المتعاقدة**، والتي تتولى تقديم المستندات الرسمية والتعريفية للفريق التقني.

### 4.3 بيانات الشركات والمكاتب `(Companies)`

- الاسم، العنوان، المدينة، الهاتف، البريد الإلكتروني، الواتساب
- الشعار (رابط صورة)، وصف الشركة
- نوع الشركة، حالة التوثيق

### 4.4 بيانات المراسلات `(Conversations & Messages)`

- مُعرِّفات المحادثة والمُشاركين
- نص الرسالة، تاريخ الإرسال، حالة القراءة

### 4.5 بيانات الطلبات `(BuyerRequests)`

- نوع العقار المطلوب، نطاق السعر، المنطقة
- تفاصيل المتطلبات، تعليقات على الطلبات

### 4.6 بيانات الاشتراكات `(Subscriptions & Invoices)`

- نوع الخطة المشترَك بها
- تواريخ بدء الاشتراك وانتهائه
- إثباتات الدفع (صور الإيصالات)
- سجل الفواتير

### 4.7 بيانات الجلسات والأمان `(UserRefreshTokens)`

- رمز التجديد (مُشفَّر)، تاريخ الانتهاء
- لا تُخزَّن كلمات المرور بأي صيغة واضحة
