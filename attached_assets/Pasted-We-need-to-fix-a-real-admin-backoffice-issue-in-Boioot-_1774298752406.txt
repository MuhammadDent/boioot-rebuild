We need to fix a real admin/backoffice issue in Boioot.

PROBLEM
Admin can view property details, change status, and delete the property,
but cannot fully edit the property/listing from the admin dashboard.

This is a system problem.
A system administrator must be able to edit any property/listing from admin backoffice.

GOAL
Implement full property edit capability for Admin from the admin dashboard
without breaking the existing owner/agent property edit flow.

REQUIREMENTS

1) ADMIN MUST BE ABLE TO EDIT ANY PROPERTY
- From admin property details modal/page, add a clear professional action:
  - "تعديل العقار" / "Edit Property"
- This action must open a full edit screen or drawer using the same validated form logic used in normal property editing, but in admin mode.
- Admin must be able to edit:
  - title
  - description
  - property type
  - listing type
  - category
  - price
  - area
  - city / district / location fields
  - features / amenities
  - advertiser / company relation if allowed by current architecture
  - images
  - publish/status fields if intended
  - SEO fields if property has them
  - any structured metadata currently supported by the listing model

- Do NOT use frontend-only bypass.
- Backend must explicitly allow Admin/SystemAdmin/Backoffice roles with proper permission to update any property.
- Keep ownership rules for normal users:
  - owner/agent can edit only their own listings
  - admin can edit any listing
- Authorization must be permission-based where possible, not hardcoded in UI only.

3) BACKEND FIX
Review and fix the update property endpoint/service:
- Check whether admin requests are currently blocked by:
  - ownership check
  - tenant/company scoping issue
  - missing role/permission mapping
  - DTO validation mismatch
  - route mismatch
- Refactor update logic so it supports:
  - Self-edit mode for listing owner
  - Admin-edit mode for system admin

4) FRONTEND FIX
In admin property details modal/page:
- Add a visible "Edit Property" button.
- Clicking it should navigate to:
  - a dedicated admin edit page

  - open a full edit modal/drawer

  - user edit flow
  - admin edit flow

5) IMAGES MUST BE EDITABLE

- see existing property images
- remove images
- upload additional images
- reorder images if feature exists

6) STATUS / SAVE UX
