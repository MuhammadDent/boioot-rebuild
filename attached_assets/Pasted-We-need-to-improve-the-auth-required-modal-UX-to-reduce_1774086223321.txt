We need to improve the auth-required modal UX to reduce friction and increase conversion.

==================================================
GOAL
==================================================

When a guest tries to perform a protected action
(Add Listing / Add Request / Send Message / Favorite / Dashboard),
do not show only a message with buttons.

Instead, show a modal that contains:
1. the auth-required message
2. a quick registration form directly inside the modal
3. a secondary option for existing users to log in

==================================================
NEW DESIRED UX
==================================================

Replace the current simple auth-required popup with a conversion-focused modal.

The modal should include:

A) Header
- icon
- title: يلزم تسجيل الدخول أو إنشاء حساب
- short explanation

B) Quick registration form directly inside the modal

Fields:
- الاسم
- البريد الإلكتروني
- رقم الجوال
- كلمة المرور
- تأكيد كلمة المرور
- نوع الحساب (optional if already known from context)

C) Primary CTA
- إنشاء حساب ومتابعة

D) Secondary CTA
- لدي حساب بالفعل / تسجيل الدخول

E) Close action
- إغلاق

==================================================
BEHAVIOR
==================================================

1. If the user submits the registration form successfully:
   - create the account
   - log them in automatically if possible
   - then continue to the original intended action

Examples:
- if they clicked "Add Request", continue to request creation
- if they clicked "Add Listing", continue to add listing flow
- if they clicked "Send Message", return them to that listing context and open message flow if possible

2. If they already have an account:
   - allow switching the modal to login mode
   - do not force them to leave the page unless necessary

==================================================
MODAL MODES
==================================================

The modal should support 2 tabs or 2 modes:

Mode 1: Register
Mode 2: Login

Default mode:
- Register

This is important because we want faster conversion for new users.

==================================================
REGISTER FORM REQUIREMENTS
==================================================

- inline validation
- loading state on submit
- clear success/error messages
- password show/hide toggle
- confirm password validation
- phone validation if supported
- RTL Arabic layout
- responsive mobile design

==================================================
TECHNICAL REQUIREMENTS
==================================================

- Build this as a reusable modal component
- Preserve current page context
- Preserve intended action after auth
- Do not force a full page redirect unless needed
- If API supports it, submit via AJAX and keep user in modal flow
- Only redirect after successful auth if the intended action requires a separate route

==================================================
IMPORTANT
==================================================

Do not remove public browsing.
Only improve the auth-required modal so it becomes a frictionless register/login entry point.

The user should not have to:
- close the modal
- go to a separate register page
- then come back manually
