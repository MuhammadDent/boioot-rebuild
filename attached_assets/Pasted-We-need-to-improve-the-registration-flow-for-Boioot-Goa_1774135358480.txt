We need to improve the registration flow for Boioot.

Goal:
When a new user creates an account, the system should collect business/location information relevant to their account type.

Business requirement:
This is a real estate SaaS platform, so location is a core part of the account identity.

Implement the registration/account setup flow professionally as follows:

1) Add account type selection during registration:
- Broker / Agent
- Real Estate Office
- Real Estate Development Company

2) After successful account creation, redirect the user to a professional onboarding/setup step
instead of forcing all fields into the first registration form.

3) In the onboarding step, collect these required business profile fields:
- Business/Office/Agent display name
- City
- District / neighborhood
- Address text
- Map location picker (preferred)
- Latitude
- Longitude
- Primary phone number
- Optional WhatsApp
- Optional description / bio

4) UX rules:
- Registration should remain simple and fast
- Business/location details should be completed immediately after first login
- Show progress UI like: Account Created → Complete Profile → Done
- Validate required fields clearly
- Do not allow publishing listings before business profile/location is completed

5) Data architecture:
- Store these fields in a proper business profile entity
- Keep it scalable for multi-tenant SaaS structure
- Support different profile types without duplicating logic
- Avoid hardcoded role-specific UI logic where possible

6) Admin/backend readiness:
- The profile should be visible later in dashboard/admin
- These fields should be editable from the dashboard profile/settings page

7) Important:
- Review the current auth flow before implementation
- Do not break existing registration/login
- Implement in a clean, production-ready way
- Follow current project architecture and naming conventions

Please first inspect the current registration/auth/profile structure,
then implement the onboarding/profile completion flow step by step.
